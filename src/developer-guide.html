Overview

The OperateID Components library provides a pure JavaScript solution for fetching and displaying blockchain identity data. It integrates with the Accumulate blockchain and Qoboto API to retrieve identity information including logos, descriptions, and background images.

What You'll Learn
  • How to include and initialize the library
  • How to fetch identity data from the blockchain
  • How to render fetched data in HTML
  • How to handle errors and edge cases

Architecture

The library follows a layered architecture pattern:

1
AdiParse API Layer - Simple, user-facing methods
2
Service Layer - OpIdService coordinates resources
3
Resource Layer - OpIdResource manages data and caching
4
API Layer - QobotoApiService fetches from blockchain
5
Utility Layer - URL parsing, formatting, network config
Design Pattern Uses Factory, Repository, and Service patterns for clean separation of concerns and testability.

Quick Start

Get started in 3 simple steps:

JavaScript - Your first OperateID integration 
// Step 1: Configure the API (optional - has smart defaults)
AdiParse.setApiBaseUrl("https://localhost:7033");
AdiParse.setDevelopmentMode(false); // Set to false for real API
AdiParse.setDebugMode(true); // Enable logging

// Step 2: Generate URLs (synchronous)
const bankUrl = AdiParse.getBankOnLedgerUrl("sunstream.acme");
const qobotoUrl = AdiParse.getQobotoUrl("sunstream.acme");
console.log(bankUrl);   // https://sunstream.BankOnLedger.com/?current-network=mainnet
console.log(qobotoUrl); // https://sunstream.Qoboto.com/?current-network=mainnet

// Step 3: Fetch data from blockchain (asynchronous)
async function loadIdentityData() {
    const data = await AdiParse.getAllMainData("sunstream.acme");
    console.log("Logo:", data.logoUrl);
    console.log("Description:", data.sectionMain1Description);
    console.log("Background:", data.sectionMain1Background2ImageUrl);
}

loadIdentityData();

Data Flow: How It Works

Understanding the Request Flow

When you call AdiParse.getAllMainData("sunstream.acme"), here's what happens:

Step-by-Step Execution Flow 
// 1. INPUT: Your code calls the API
await AdiParse.getAllMainData("sunstream.acme")

// 2. FORMATTING: Identity URL is formatted
// "sunstream.acme" → "acc://sunstream.acme"
identityFormatter.formatIdentity()

// 3. RESOURCE CREATION: OpIdResource is created with parsed identity
opIdResourceFactory.createByIdentityUrl()
  → opIdInfoFactory.createByIdentityUrl()
    → Parse: acc://sunstream.acme → name="sunstream"

// 4. API CALL: HTTP request to Qoboto API
GET https://localhost:7033/api/v1/Qoboto/GetDataValue/All?DataAccountUrl=sunstream.acme

// 5. RESPONSE PARSING: API returns array of sections
[
  { name: "header", ... },
  { name: "main", logoUrl: "...", sectionMain1Description: "...", ... },
  { name: "footer", ... }
]

// 6. SECTION EXTRACTION: Find object with name="main"
_findMainSection() → returns main section object

// 7. TEXT CLEANING: Remove problematic characters
_cleanText() → removes "�" characters, fixes encoding

// 8. CACHING: Store result for 5 minutes
cache.set("qoboto_sunstream.acme", { data, timestamp })

// 9. OUTPUT: Return cleaned data to your code
{
  logoUrl: "https://example.com/logo.png",
  sectionMain1Description: "Welcome to Sunstream...",
  sectionMain1Background2ImageUrl: "https://example.com/bg.jpg",
  rawData: { ... }
}

API Reference

Core Methods

Method Type Returns Description
getBankOnLedgerUrl(identityUrl) Sync string | null Generate BankOnLedger URL
getQobotoUrl(identityUrl) Sync string | null Generate Qoboto URL
getLogoUrl(identityUrl) Async Promise<string | null> Fetch logo URL from API
getSectionMain1Description(identityUrl) Async Promise<string | null> Fetch description from API
getSectionMain1Background2ImageUrl(identityUrl) Async Promise<string | null> Fetch background image URL
getAllMainData(identityUrl) Async Promise<object> Fetch all data in one call (efficient)

Fetching Data from Blockchain

Method 1: Individual Calls

JavaScript - Fetch each piece separately 
async function fetchIndividualData(identityUrl) {
    try {
        // Fetch logo
        const logoUrl = await AdiParse.getLogoUrl(identityUrl);
        console.log("Logo:", logoUrl);

        // Fetch description
        const description = await AdiParse.getSectionMain1Description(identityUrl);
        console.log("Description:", description);

        // Fetch background image
        const bgImage = await AdiParse.getSectionMain1Background2ImageUrl(identityUrl);
        console.log("Background:", bgImage);

    } catch (error) {
        console.error("Error fetching data:", error);
    }
}

fetchIndividualData("sunstream.acme");

Method 2: Batch Call (Recommended)

JavaScript - Fetch all data in one API call 
async function fetchAllData(identityUrl) {
    try {
        // Single API call - much more efficient!
        const data = await AdiParse.getAllMainData(identityUrl);

        // All data is now available
        console.log("Logo:", data.logoUrl);
        console.log("Description:", data.sectionMain1Description);
        console.log("Background:", data.sectionMain1Background2ImageUrl);
        console.log("Raw API Data:", data.rawData);

        return data;

    } catch (error) {
        console.error("Error fetching all data:", error);
        return null;
    }
}

fetchAllData("sunstream.acme");
Performance Tip Always use getAllMainData() instead of individual calls. It makes only ONE API request and is cached for 5 minutes.

Rendering Data in HTML

Step-by-Step Rendering Process

JavaScript - Complete fetch and render example 
async function displayIdentityCard(identityUrl) {
    // 1. Get container element
    const container = document.getElementById('identity-container');

    // 2. Show loading state
    container.innerHTML = '<div class="loading">Loading...</div>';

    try {
        // 3. Fetch all data from blockchain
        const data = await AdiParse.getAllMainData(identityUrl);

        // 4. Generate URLs
        const bankUrl = AdiParse.getBankOnLedgerUrl(identityUrl);
        const qobotoUrl = AdiParse.getQobotoUrl(identityUrl);

        // 5. Extract identity name
        const identityName = identityUrl.split('.')[0].replace('acc://', '');

        // 6. Build HTML with fetched data
        const html = `
            <div class="identity-card">
                <!-- Logo Section -->
                <div class="card-header">
                    <img src="${data.logoUrl}"
                         alt="${identityName} logo"
                         onerror="this.src='fallback-logo.png'">
                    <h2>${identityName}</h2>
                </div>

                <!-- Links Section -->
                <div class="card-links">
                    <a href="${qobotoUrl}" target="_blank">
                        Visit Qoboto Site
                    </a>
                    <a href="${bankUrl}" target="_blank">
                        Bank on Ledger
                    </a>
                </div>

                <!-- Background Image Section -->
                <div class="card-background"
                     style="background-image: url('${data.sectionMain1Background2ImageUrl}')">
                </div>

                <!-- Description Section -->
                <div class="card-description">
                    <p>${data.sectionMain1Description}</p>
                </div>
            </div>
        `;

        // 7. Inject HTML into DOM
        container.innerHTML = html;

    } catch (error) {
        // 8. Handle errors gracefully
        container.innerHTML = `
            <div class="error">
                Failed to load identity data: ${error.message}
            </div>
        `;
        console.error("Rendering error:", error);
    }
}

// Call the function
displayIdentityCard("sunstream.acme");

Dynamic Content Updates

JavaScript - Update specific elements 
async function updateIdentityElements(identityUrl) {
    const data = await AdiParse.getAllMainData(identityUrl);

    // Update individual DOM elements
    document.getElementById('logo').src = data.logoUrl;
    document.getElementById('description').textContent = data.sectionMain1Description;

    const qobotoUrl = AdiParse.getQobotoUrl(identityUrl);

    document.getElementById('bank-link').href = bankUrl;
    document.getElementById('qoboto-link').href = qobotoUrl;
}

Best Practices

1. Error Handling

JavaScript - Robust error handling 
async function safeDataFetch(identityUrl) {
    try {
        const data = await AdiParse.getAllMainData(identityUrl);

        // Check if data is valid
        if (!data || !data.logoUrl) {
            throw new Error("Invalid or incomplete data received");
        }

        return data;

    } catch (error) {
        console.error("Data fetch failed:", error);

        // Return fallback data
        return {
            logoUrl: "default-logo.png",
            sectionMain1Description: "Identity data unavailable",
            sectionMain1Background2ImageUrl: "default-bg.jpg"
        };
    }
}

2. Performance Optimization

Best Practices
  • Use getAllMainData() instead of multiple individual calls
  • Data is automatically cached for 5 minutes
  • Enable setDevelopmentMode(true) during testing to avoid API rate limits
  • Use clearCache() sparingly - let the automatic cache work

3. Network Configuration

JavaScript - Multi-network support 
// Allow users to switch networks
function switchNetwork(networkName) {

    // URLs automatically update

    // Kermit:  https://sunstream.BankOnLedger.com/?current-network=kermit
}

Troubleshooting

Common Issues and Solutions

❌ Problem: "AdiParse is not defined"

Solution: Make sure operateIdAssembly.js is loaded before your script. Move your code inside a window.onload or DOMContentLoaded handler.

❌ Problem: API returns null or undefined

Solution:

  • Enable debug mode: AdiParse.setDebugMode(true)
  • Check browser console for API errors
  • Verify API URL is correct
  • Check CORS settings on your API server
❌ Problem: Special characters (�) in description

Solution: Already handled! The library automatically cleans problematic characters using _cleanText() method.
